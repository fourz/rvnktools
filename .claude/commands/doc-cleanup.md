# Document Cleanup & Organization

Maintain documentation standards, verify knowledge base synchronization, and archive deprecated files.

**Workflow Type**: Recurring (Monthly)
**Default**: `audit` (check without changes)

---

## Actions

- **audit** - Check current state, report findings
- **organize** - Rename and move files to correct locations
- **archive** - Move old docs to archive/
- **validate** - Verify standards compliance

---

## Documentation Categories

| Category | Location | Sync? | Purpose |
|----------|----------|-------|---------|
| **Spec** | docs/spec/ | ✅ YES | Specifications, architecture |
| **Guide** | docs/guide/ | ✅ YES | Tutorials, setup |
| **API** | docs/api/ | ✅ YES | API documentation |
| **Reference** | docs/reference/ | ✅ YES | Configuration, CLI, operational guides |
| **Operations** | merged → reference/ | — | Merged into reference/ |
| **Fix** | docs/fix/ | ❌ NO | Temporal: bug fixes |
| **Release** | docs/release/ | ⚠️ Limited | Release notes, version documentation (Ravenkraft projects) |
| **Milestones** | docs/milestones/ | ❌ NO | Milestone reports (Sonnet-generated documentation) |
| **Archive** | docs/archive/ | ❌ NO | Deprecated docs |
| **Summary** | docs/summary/ | ✅ Optional | Audit reports, cleanup summaries |

**Naming**: Long-term `kebab-case.md`, Temporal `YYYY-MM-DD-description.md`

---

## Special Purpose Directories

### operations/ → reference/

**Migration**: Move contents to `docs/reference/`
**Purpose**: Operational guides and procedures (e.g., archon backup/update strategies)
**Status**: These are reference materials, not temporal
**Sync**: ✅ YES (moved to reference/)

**Action**: Consolidate operational content into reference/ with naming like:

- `archon-backup-strategy.md` (in reference/)
- `archon-update-procedures.md` (in reference/)

### release/

**Location**: `docs/release/`
**Purpose**: Release notes, version documentation, deployment info for Ravenkraft projects
**Content**: Version-specific notes, changelog entries, breaking changes, migration guides
**Status**: ✅ Official release documentation
**Sync**: ⚠️ Limited (version-specific, may archive old releases)

**Naming**: Version-based or changelog-based

- `changelog.md` (consolidated)
- `v1-0-0-release-notes.md`
- `migration-to-v2.md`

### milestones/

**Location**: `docs/milestones/`
**Purpose**: Milestone completion reports (typically verbose documentation generated by Sonnet)
**Content**: Project milestone achievements, comprehensive status updates
**Status**: Project/milestone tracking, not active operations
**Sync**: ❌ NO (milestone reports are point-in-time, not persistent knowledge)

**Usage**: Archive old milestones, keep only current/relevant ones

- Decision: Keep for project history or archive older than 3 months?

### repos/

**Location**: `docs/repos/`
**Status**: ❌ UNUSED - Empty stub directories (likely auto-generated)
**Recommendation**: Delete or clarify purpose

- Currently contains: `RVNKLore/target/`, `RVNKQuests/target/`, `RVNKWorlds/target/`
- Appears to be orphaned build artifacts
- **Action**: Remove unless actively used for repository-specific docs

## Audit Checklist

1. ✅ Correct location per category
2. ✅ Naming convention compliance
3. ✅ Content still relevant
4. ✅ Synced to knowledge base (if long-term)
5. ✅ Cross-references valid

---

## Common Operations

**Scan**: `find docs -type f -name "*.md" | sort`

**Move with history**: `git mv docs/old.md docs/spec/new.md`

**Archive**: `git mv docs/spec/deprecated.md docs/archive/`

**Verify sync**: `rag_search_knowledge_base(query="concept", match_count=3)`

**Component check** (Claude Code): `ls .claude/{agents,skills,commands}/*.md | wc -l`

**Full validation**: See [component-management.md](../skills/component-management.md)


## Best Practices

✅ Use `git mv` (preserves history)
✅ Update cross-references before committing
✅ Document archival reasons

❌ Copy + delete (loses history)
❌ Skip cross-reference updates
